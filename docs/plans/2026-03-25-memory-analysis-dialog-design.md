# Redis Desktop Memory Analysis Dialog Design

**日期**

- 2026-03-25

**背景**

- 当前内存分析弹窗的标题、筛选表单和主操作按钮都堆在同一层横向布局里。
- 在较窄窗口或浅色主题下，`Scan` 按钮会和最小值输入框互相挤压，右上角关闭入口也不够明显。
- 用户需要一个更稳定的桌面弹窗布局，并希望右上角有和“新建连接”一致的叉叉关闭图标。

**目标**

- 将内存分析弹窗重排为更清晰的“头部 + 筛选区 + 状态/结果区”结构。
- 让筛选输入与扫描按钮在不同宽度下都保持稳定对齐，不再出现内容错乱。
- 强化右上角关闭按钮的可见性和点击热区，同时复用现有通用弹窗交互。

**结论**

- 保留 `AnimatedDialog` 作为通用容器，不新增专用弹窗基础组件。
- 将内存分析弹窗顶部改为结构化标题区，加入简短说明文案和更清晰的筛选卡片。
- 调整通用关闭按钮样式，显式传递 `IconX` 颜色并增加背景、边框和点击区域，使其在浅色/深色主题下都稳定可见。

## 一、方案选择

- 不只修修单行表单的宽度。那只能缓解局部错位，无法解决标题区和工具区层级混乱的问题。
- 不重写扫描逻辑、结果表格逻辑或数据流。本次只处理布局和交互可见性，降低风险。
- 采用局部结构重排：头部说明、筛选卡片、状态条、结果表格各自成块，桌面端保持两列输入，小宽度下自然换行。

## 二、实现策略

`src/ui/memory_analysis_dialog.rs`

- 新增头部区域，标题左侧保留搜索图标，右侧留出关闭按钮安全空间。
- 将筛选区改成卡片式容器：`Pattern` 和 `Min Size` 独立字段，操作按钮移到单独一行。
- 优化空状态、扫描状态、错误状态和结果摘要条的间距、边框和文字层级。
- 保留点击结果项后选中 key 并关闭弹窗的现有行为。

`src/ui/animated_dialog.rs`

- 保留用户当前已经加入的聚焦逻辑。
- 调整右上角关闭按钮的尺寸、背景、边框和图标显式颜色。
- 给对话框主体增加更充足的右上角安全留白，避免按钮被标题内容“吃掉”。

## 三、验收标准

- 弹窗打开后，标题、筛选输入框和扫描按钮不再互相挤压。
- 右上角关闭按钮在浅色和深色主题下都清晰可见，点击可关闭弹窗。
- 扫描中、空结果、错误和结果表格四种状态仍可正常展示。
- `cargo check` 通过，无新增编译错误。
