# Redis Engine Key-List-First Layout Design

**日期**

- 2026-03-22

**问题**

- 当前 `Data` 页默认采用固定上下分栏。
- 上半区 `KeyBrowser` 默认高度过低，下半区在未选中 key 时还会直接显示 `ServerInfoPanel`。
- 结果是高频区 `Key 列表` 被低频区 `服务器概览` 抢占空间，用户需要先拖动分割线，才能勉强看到少量 key。

**目标**

- 默认把主工作区优先让给 key 列表。
- 只有在用户明确需要“编辑 key”或“查看服务器概览”时，详情区才展开。
- 让列表区在常规窗口高度下稳定显示足够多的行，避免需要先调布局再开始使用。

## 一、总体策略

采用“列表优先 + 详情按需展开”的布局：

1. 默认态：
   - Key 列表占据主工作区。
   - 底部只保留一条紧凑状态条。
2. 选中 key：
   - 自动展开详情编辑区。
   - 编辑区高度受控，不允许压缩列表到不可用。
3. 未选中 key 且用户主动查看服务器概览：
   - 展开服务器概览区。
4. 未选中 key 且用户未主动查看概览：
   - 不渲染大块 `ServerInfoPanel`。

## 二、交互状态

定义三种视图状态：

- `Collapsed`
  - 默认状态。
  - 展示底部状态条，提示“选择一个 Key 查看详情”。
- `Editor`
  - 用户点击 key 后进入。
  - 下半区显示 `ValueViewer`。
- `Overview`
  - 用户主动点击“服务器概览”后进入。
  - 下半区显示 `ServerInfoPanel`。

状态切换规则：

- 点击 key：进入 `Editor`
- 收起详情：回到 `Collapsed`
- 点击服务器概览：进入 `Overview`
- 在 `Overview` 中点击 key：切到 `Editor`

## 三、空间分配

- `KeyBrowser` 不再依赖固定像素高度作为默认主布局。
- 主区域改为：
  - 列表区 `flex: 1`
  - 详情区固定高度，可拖动
- 详情区默认高度控制在 280-320px。
- 详情区最小高度约 220px。
- 列表区最小高度提高，确保常规窗口下能稳定看到 8-12 行 key。

## 四、服务器概览调整

- `ServerInfoPanel` 从“默认常驻下半区”改为“按需展开内容”。
- 默认状态只显示简短摘要，不显示完整统计卡片网格。
- 这样可以把空间还给高频操作区域。

## 五、状态条设计

底部状态条承担三个角色：

- 当前上下文提示
- 快捷操作入口
- 详情区状态切换入口

建议内容：

- 未选中 key：
  - “选择一个 Key 查看详情”
  - “服务器概览”按钮
- 已选中但折叠：
  - 显示当前选中 key
  - “展开详情”按钮
- 详情展开中：
  - 显示当前模式（编辑器 / 概览）
  - “收起详情”
  - “恢复默认高度”

## 六、非目标

- 本次不重写 `ValueViewer` 各类型编辑器。
- 本次不重写 `ServerInfoPanel` 数据来源。
- 本次只调整 `Data` 页主工作区布局与交互优先级。
