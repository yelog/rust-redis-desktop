# Redis Engine Light Theme Design

**日期**

- 2026-03-22

**背景**

- 当前暗色模式已经建立了明确的产品气质：深底、暖红主色、青色强调、高对比信息密度。
- 当前亮色模式仍然沿用偏系统默认的蓝灰体系，导致亮暗模式在品牌识别、层级语义和交互反馈上像两套产品。

**目标**

- 让亮色模式继承暗色模式的品牌语义，而不是继续使用系统蓝灰。
- 保持数据工具界面的专业感、可读性和高信息密度。
- 尽量只改 theme token 与亮色派生色，减少组件级局部特判。

**结论**

- 亮色模式采用“暖白底 + 暖灰层级 + 深珊瑚主色 + 深青色强调”。
- 暗色模式中承担“品牌感”的 `primary` 与“操作/焦点感”的 `accent` 在亮色模式继续保留同样分工。
- 亮色模式不追求纯白和系统蓝；背景应更柔和，边框更暖，文本层级更稳定。

## 一、配色原则

- 背景从冷白改为暖白，降低刺眼感，让亮色界面更接近暗色模式的材质感。
- 主色从系统蓝改为深珊瑚红，承接暗色模式的品牌按钮和标题高光。
- 强调色从系统蓝改为深青色，承接暗色模式里的 scan、选中态和焦点色。
- 文本和边框统一带轻微暖灰，不再使用纯中性灰。

## 二、Token 调整方向

`src/theme/colors.rs`

- `background`: 冷灰白 -> 暖白
- `background_secondary`: 纯白 -> 柔和暖白
- `background_tertiary`: 生硬浅灰 -> 暖灰卡片面
- `surface_*`: 围绕暖白和暖灰建立 4 层台阶
- `border` / `outline_variant`: 从冷灰边框改为偏暖的描边
- `text_secondary` / `text_subtle`: 从通用灰改为暖灰文本
- `primary`: 系统蓝 -> 深珊瑚红
- `accent`: 系统蓝 -> 深青色

## 三、派生色同步

`src/ui/app.rs`

- `controlBg`、`buttonSecondary`、`selectionBg`、`infoBg`、`successBg`、`errorBg` 需要和新的亮色 token 同步。
- 语法高亮色不再走 GitHub 风格蓝绿紫，改为更接近当前产品的暖红 / 青绿 / 琥珀组合。

## 四、验收标准

- 亮色模式下，按钮、选中态、搜索输入、表格 hover、标签徽标都应与暗色模式有明显“同一家产品”的感觉。
- 文本对比度保持可读，不出现浅灰字压在暖白底上发虚的问题。
- 切换主题时，不再出现“暗色是 Redis Engine，亮色像默认组件库”的割裂感。
